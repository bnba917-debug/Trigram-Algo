# 动画重构 — 手动验证 Checklist

> 由于前端特效（Three.js 渲染、CSS 动画、Canvas 粒子）无法用传统单元测试覆盖，本文档提供**手动验证步骤**，作为 TDD 的替代方案。

**验证环境**：Chrome 最新版 + 移动端 Safari/Chrome（iPhone 12+ / Android 10+）

---

## Phase 1: 视觉层级清理

### P1-1 关闭 star-rays 层
**操作**：点击"启坛引星辰"，观察起卦动画期间是否有旋转的星芒效果
**预期**：
- [ ] 起卦期间**无**旋转星芒（conic-gradient 效果）
- [ ] 画面更暗、更聚焦于中心卦环

### P1-2 弱化 mysticAmbience
**操作**：进入首页，观察背景层
**预期**：
- [ ] 无漂浮的卦符（☰☱☲ 等）
- [ ] 无随机噪点颗粒
- [ ] 只剩轻微的雾气效果（3 个渐变圆）

### P1-3 数符雨边缘分布
**操作**：点击"启坛引星辰"，观察数符雨
**预期**：
- [ ] 雨滴只出现在屏幕左右边缘（约各占 20% 宽度）
- [ ] 中间区域（60% 宽度）无雨滴
- [ ] 雨滴数量比之前少，分布更自然

### P1-4 起卦时星场变暗
**操作**：点击"启坛引星辰"，观察 Three.js 星场
**预期**：
- [ ] 起卦期间星场（金色/紫色光点）明显变暗（不透明度 ~0.25）
- [ ] 卦环（HTML 层）成为视觉焦点

### P1-5 数符雨 z-index 调整
**操作**：起卦时观察数符雨与卦环的层叠关系
**预期**：
- [ ] 数符雨在卦环**下方**（不会遮挡 01·乾 等标签）

### P1-6 星辰灌注流星减半
**操作**：起卦期间观察 starPower 流星
**预期**：
- [ ] 同时出现的流星数量 ≤ 6 颗
- [ ] 流星轨迹更清晰，不密集重叠

## Phase 2: 起卦节奏重构

### P2-1 总时长压缩
**操作**：用秒表计时，从点击按钮到结果 UI 出现
**预期**：
- [ ] 总时长 ≤ 6 秒（之前约 8.5 秒）

### P2-2 星光聚拢后停顿
**操作**：起卦时观察星光向心聚拢动画
**预期**：
- [ ] 聚拢动画结束后，有 **0.6 秒** 明显静止（星场/粒子暂停运动）
- [ ] 停顿后才开始冲击波

### P2-3 星流只触发一次
**操作**：起卦期间观察星流效果
**预期**：
- [ ] 整个起卦过程只出现**一次**星流（emitStarStreamsFn）

### P2-4 相机运动克制
**操作**：起卦时观察画面整体缩放感
**预期**：
- [ ] 相机拉近/拉远幅度明显小于之前（视觉上更平稳）

### P2-5 粒子爆炸范围缩小
**操作**：起卦时观察金色粒子爆炸
**预期**：
- [ ] 粒子数量比之前少
- [ ] 爆炸范围更集中（不扩散到屏幕边缘）

### P2-6 代码闪现后留白
**操作**：起卦时观察股票代码闪现
**预期**：
- [ ] 股票代码闪现后，有 **0.4 秒** 停顿
- [ ] 停顿后才显示"伯温天机"结果区域

### P2-7 冲击波更柔和
**操作**：起卦时观察金光冲击波
**预期**：
- [ ] 冲击波颜色更柔和（不刺眼）
- [ ] 不透明度降低，视觉冲击减弱

## Phase 3: 卦环仪式状态 + 移动端

### P3-1 is-ritual 视觉反馈
**操作**：点击"启坛引星辰"，观察卦环标签
**预期**：
- [ ] 起卦期间，卦环标签边框出现**金光呼吸效果**
- [ ] 序号颜色临时提亮（从 #ffcc44 变为更亮）
- [ ] 整体视觉上"亮起来"

### P3-2 起卦时卦环半透明
**操作**：起卦期间观察卦环
**预期**：
- [ ] 卦环整体不透明度降至 ~0.55
- [ ] 结束时恢复为 1.0

### P3-3 起卦时卦环旋转变慢
**操作**：起卦期间观察卦环旋转速度
**预期**：
- [ ] 旋转速度比平时慢约 30%

### P3-4 小屏流星增加
**操作**：在手机上点击"启坛引星辰"
**预期**：
- [ ] 手机上流星数量比桌面版多 4 颗

### P3-5 小屏卦名字号
**操作**：在手机上观察卦环标签
**预期**：
- [ ] "01·乾" 类标签字号 ≥ 12px（不会太小看不清）
